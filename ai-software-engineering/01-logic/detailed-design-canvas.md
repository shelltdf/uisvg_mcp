# 详细设计：画布（网格 / 缩放 / 平移）

## 坐标系

- **文档坐标**：SVG 用户空间，由根 `viewBox` / 内容决定；**逻辑画布原点 (0,0) 为左上角**（与编辑器视口内逻辑画布左上角一致）。
- **视图变换**：在宿主层对“画布内容组”应用 CSS `transform: translate(tx, ty) scale(s)`（或等价矩阵），不改变磁盘上 SVG 的坐标，仅改变显示。
- **视图保持**：在同一编辑会话中，因添加图元、组件等导致 `svgMarkup` 更新时，**不**自动重置平移与缩放；**新建文档**、**打开文件**、**重置视图** 时显式恢复默认观察。

## 网格

- 在画布背景层绘制正交网格（例如步长 16，与 `uisvg:canvas/@grid` 对齐）。
- 网格随 `scale` 缩放；可选在缩放低于阈值时隐藏细线。

## 逻辑画布边界与标注

- 可编辑区域尺寸与 `metadata` 中 **`uisvg:canvas`** 的 `width`/`height` 及根 `svg` 的 `viewBox` 一致；**`dpi`** 为文档/导出参考（用户单位仍为 px）。
- **白底、网格与左上角说明** 仅在编辑器内以 HTML/CSS 叠层绘制，**不**序列化进 `.ui.svg`。
- **视图 → 画布设置…** 可修改宽高、网格、DPI，并写回 `uisvg:canvas` 与根 `svg` 属性。

## 输入

| 操作 | 行为 |
|------|------|
| 滚轮 | 以指针位置为锚点缩放（无需 Ctrl）；在左下角对齐调试面板内滚动时拦截冒泡，避免误缩放画布 |
| 中键拖拽 | 平移视图 |
| 左键点击 | 选择：从命中目标向上查找最近带 `id` 的 SVG 元素，同步当前选中（用于属性等）；**Shift+点击** 追加/切换多选 |
| 左键空白拖拽 | **框选**：与逻辑画布矩形求交，命中多个 uisvg 对象根则多选 |
| 左键拖拽（已选中对象上） | 平移被选中对象根（支持多选一并平移） |
| （可选）空格 + 左键拖拽 | 平移 |

## 选中框缩放与控件 Chrome

- 选中 uisvg 对象根后，选中框缩放手柄调整的是对象内 **主几何元素**（实现：`getResizeTargetElement`，常为带 `data-uisvg-part` 的 `*-face` / 槽 `rect` 等）。
- 缩放操作结束后，画布对对象根调用 **`ensureResizeChromeLayoutSynced`**，按控件类型 **重算** 标题栏、列头线、表格网格线、固定厚度滚动条/滑块槽等，使 WinForms 占位在尺寸变化后仍保持可读比例，而非简单拉伸所有子节点。
- 字段级规则、各控件 `data-uisvg-part` 命名与常量以 **`02-physical/uisvg-editor-web/spec.md`**（画布表「选中框缩放」「控件 Chrome 重布局」）及 **`svgElementResize.ts`** / **`libraryPlacement.ts`** 为准。

## 多选、剪贴板与对齐

- **删除 / 复制 / 粘贴 / 对齐**：画布 **右键菜单** 顺序为 **复制 → 粘贴 →（分隔）→ 对齐子菜单 →（分隔）→ 删除**（删除在末位）；菜单 **编辑** 下亦有删除/复制/粘贴与六种对齐；工具栏提供 **六个对齐图标**（未选满 2 个可移动对象时为禁用态）。
- **对齐实现**：对当前文档字符串 `DOMParser` 解析，在内存文档上改 `transform`/`x`/`y` 等，再写回 `svgMarkup`。**不得**依赖挂屏布局矩阵：对 `ownerDocument !== window.document` 的节点优先用 **`transform` 父链** 累积到根 SVG 用户坐标；`SVGMatrix` 转 `DOMMatrix` 时用 **`[a,b,c,d,e,f]`** 数组构造，避免 `new DOMMatrix(svgMatrix)` 抛错。
- **对齐调试**：左下角面板 **仅显示对齐摘要**（由 `alignUisvgObjectRoots` 写入）；通过 **视图 / 工具栏调试图标** 开关显示；与选中框几何、框选过程解耦。

## 与 uisvg 关系

- 新建文档时写入默认 `uisvg:canvas`（含 `dpi`）与空 `layer-root`；保存时回写当前视口偏好（可选后续）。
